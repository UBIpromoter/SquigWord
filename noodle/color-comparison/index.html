SquigWord Analysis

How Our Color Rendering Differs from Snowfro's

The Color Formula

The actual math for picking colors is identical. Both use the same HSB cycle, the same counter, the same seed-derived parameters. If you froze a single point in space, it would be the same color in both systems.

Snowfro hue = reverse ? 255 - (((color / spread) + startColor) % 255) : (((color / spread) + startColor) % 255) colorCounter++ // once per circle drawn
SquigWord v2.4 hue = reverse ? 255 - (((color / spread) + startColor) % 255) : (((color / spread) + startColor) % 255) color++ // once per circle drawn
Identical
The Structural Difference

This is the root cause of every visual gap. It's not about the color formula — it's about how many circles get drawn and where they land.

Snowfro One continuous wave. 12-20 segments (hash-driven). 50-200 steps per segment. Slinky example: ~15 segments × 50 steps = ~750 evenly-spaced rings along one smooth curve.
SquigWord v2.4 Many short letter strokes. Each letter = separate path. 3-15 control points per letter. 8-10 steps per segment. "Squiggles" example: ~10 paths × ~8 segs × 10 steps = ~80-100 rings per stroke but broken across many paths.
Structural — can't fully close
Snowfro draws one long rope with hundreds of circles. We draw many short ropes with fewer circles each. This means our rings are sparser, our color transitions are choppier, and our endpoint dots are rarer. Every type-specific gap traces back to this.
Step Counts

Step count controls how many circles land between each pair of control points. Higher = denser rings = smoother color gradient. But our paths are shorter, so the same step count produces fewer total circles.

We reduced step counts because our letter paths have fewer segments and higher step counts were producing too-dense output at the letter scale. The trade-off: sparser rings, less smooth gradients, fewer filled endpoint caps.

Pipe — The Biggest Gap

Pipe is where the visual difference is most obvious. Three specific issues compound:

Snowfro's Pipe // Every point gets both rings: // 1. Black outer — STROKED arc(x, y, (H/7) / 2) fill at segment endpoints → i===0 || i===steps-1 → ~30 filled dots per wave stroke: black, weight: sw // 2. Colored inner — STROKED arc(x, y, (H/13) / 2) fill at segment endpoints stroke: colored, weight: sw Both use same hairline weight. Filled dots at every segment join = colored dots with black borders scattered all along the wave.
SquigWord's Pipe // Every point gets both rings: // 1. Black outer — STROKED arc(x, y, (eH/5) / 2) fill at path endpoints only → first & last point per stroke → ~2 filled dots per letter stroke: black, weight: sw×1.3 // 2. Colored inner — STROKED arc(x, y, (eH/13) / 2) fill at path endpoints only stroke: colored, weight: slinkySw → max(sw×2, 1.0px) Different stroke weights. Filled dots only at stroke tips = far fewer colored-dot-in-black moments.
The "colored dots surrounded by black" effect comes from Snowfro filling at segment boundaries — which happen every 50 steps along a 750-point wave (~15 times). We only fill at path endpoints — 2 per letter stroke. That's the single biggest visual difference in Pipe rendering.
Stroke Weight

A subtle but cumulative difference.

Snowfro sw = height / 1200 One weight for everything. Black rings: sw Colored rings: sw ~0.5px hairline at 600px canvas.
SquigWord v2.4 sw = effectiveH / 1200 slinkySw = max(sw × 2, 1.0) Two different weights. Black rings: sw + (slinkySw-sw)×0.3 Colored rings: slinkySw The 1.0px floor makes colored rings thicker on smaller canvases.
Our colored rings are heavier
Segment Join Double-Draw

We added a fix that Snowfro doesn't have — and it changes the visual texture.

Snowfro for i = 0 to steps: draw circle No skip. The last point of segment N overlaps the first point of segment N+1. This creates a slightly darker ring at every segment boundary — an artifact that becomes part of the texture.
SquigWord v2.4 for i = 0 to steps: if (i===steps && !lastSeg) skip draw circle We skip the overlap to prevent double-draw. Cleaner rings, but we lose the visual pulse that gives Snowfro's slinky/pipe its subtle rhythm.
Intentional divergence — debatable
What To Fix (Priority Order)
  • 1 Pipe outer size: eH/5 → eH/7. Match the Snowfro ratio. Our black rings are currently 40% too large.
  • 2 Pipe filled dots: fire at segment boundaries, not just path endpoints. Change isEndpoint to include i===0 || i===steps for each segment. This creates the scattered colored-dot-in-black effect.
  • 3 Stroke weight: use sw for everything. Drop slinkySw. Both ring colors should use the same hairline weight. Remove the 1.0px floor.
  • 4 Consider removing the segment-join skip. The double-draw at joins is part of Snowfro's visual texture. Removing it made our output cleaner but less authentic.
